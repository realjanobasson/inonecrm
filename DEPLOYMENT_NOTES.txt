WORKERS
-------
A) inonecrm-api
   - Folder: workers/inonecrm-api
   - Recommended domain: api.inonecrm.com
   - Bindings:
       D1:  DB    -> inone-db
       R2:  MEDIA -> inone-media
   - Secrets:
       JWT_SECRET (required)
       SEED_TOKEN (optional, one-time superadmin seed)

   Database migrations:
   1) wrangler d1 create inone-db
   2) Put database_id into workers/inonecrm-api/wrangler.toml
   3) cd workers/inonecrm-api
      wrangler d1 migrations apply inone-db --remote
   4) wrangler secret put JWT_SECRET
   5) wrangler deploy

B) inonecrm-realtime (optional)
   - Folder: workers/inonecrm-realtime
   - Durable Objects binding: CHAT_HUB
   - Deploy: cd workers/inonecrm-realtime && wrangler deploy

C) inonecrm-automations (optional)
   - Folder: workers/inonecrm-automations
   - Deploy: cd workers/inonecrm-automations && wrangler deploy
   - Add Cron triggers in Cloudflare (if using sequences/SLAs)

D) inonecrm-webhooks (optional)
   - Folder: workers/inonecrm-webhooks
   - Recommended domain: hooks.inonecrm.com
   - Deploy: cd workers/inonecrm-webhooks && wrangler deploy
   - Later: set Stripe/WhatsApp secrets and forward jobs to API/Queue

SUPERADMIN SEED (ADMIN DASHBOARD)
---------------------------------
1) Set SEED_TOKEN on inonecrm-api Worker.
2) Open admin app -> Seed Superadmin screen.
3) Create first superadmin.
4) Remove SEED_TOKEN afterwards.
